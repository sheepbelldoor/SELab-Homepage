# 연구실 홈페이지 기획서 초안

## 1. 프로젝트 개요

### 1.1 프로젝트명

연구실 홈페이지 구축 및 운영 체계 설계

### 1.2 추진 배경

기존 연구실 정보는 분산되어 있거나 정적으로 관리되는 경우가 많아, 외부 방문자에게 연구실의 정체성과 성과를 효과적으로 전달하기 어렵다. 또한 공지사항, 연구 성과, 구성원 정보 등의 갱신이 비정기적으로 이루어질 경우 홈페이지의 신뢰도와 활용성이 저하될 수 있다. 이에 따라, 공개용 홈페이지와 관리자 전용 관리 페이지를 분리한 형태의 연구실 홈페이지를 구축하여 정보 전달력과 운영 편의성을 동시에 확보하고자 한다.

### 1.3 추진 목적

* 연구실 소개, 연구 분야, 구성원, 논문 및 프로젝트 정보를 체계적으로 제공한다.
* 외부 방문자, 지원자, 협력 기관 등이 필요한 정보를 빠르게 확인할 수 있도록 한다.
* 관리자 1인이 중심이 되어 공지 및 주요 정보를 손쉽게 갱신할 수 있는 운영 구조를 마련한다.
* 향후 필요 시 메뉴 및 기능을 확장할 수 있는 구조를 기반으로 설계한다.

---

## 2. 구축 방향

### 2.1 기본 방향

본 홈페이지는 다음의 두 영역으로 구성한다.

1. **공개 홈페이지**
   누구나 로그인 없이 열람 가능한 영역으로, 연구실 정보 제공을 목적으로 한다.

2. **관리자 페이지**
   관리자만 로그인하여 접근 가능한 영역으로, 게시글 작성 및 주요 정보 수정 기능을 제공한다.

### 2.2 운영 원칙

* 일반 방문자는 로그인 없이 홈페이지를 열람한다.
* 관리자만 별도 URL을 통해 관리자 페이지에 접속한다.
* 게시글 및 주요 정보 수정 권한은 관리자 계정에만 부여한다.
* 운영 복잡도를 줄이기 위해 초기에는 최소 기능 중심으로 구성한다.

## 5. 공개 홈페이지 상세 구성

### 5.1 Home

첫 화면에서 연구실의 핵심 정보가 한눈에 보이도록 구성한다.

주요 구성 요소:

* 연구실 한 줄 소개
* 대표 연구 분야 키워드
* 대표 이미지 또는 배너
* 최신 소식 요약
* 대표 논문 또는 프로젝트
* Join Us / Contact 바로가기

### 5.2 About

연구실의 목표와 정체성을 소개하는 페이지이다.

주요 구성 요소:

* 연구실 소개
* 비전 및 목표
* 연구 철학 또는 운영 방향
* 위치 및 기본 정보

### 5.3 People

연구실 구성원을 소개하는 페이지이다.

하위 구분 예시:

* Professor
* Researchers / Postdocs
* PhD Students
* MS Students
* Intern / Undergraduate
* Alumni

표시 정보 예시:

* 이름
* 사진
* 직위
* 관심 연구 분야
* 이메일
* 개인 홈페이지 / GitHub / Google Scholar 링크

### 5.4 Research

연구실의 주요 연구 분야를 정리하는 페이지이다.

주요 구성 요소:

* 연구 분야 개요
* 세부 연구 주제
* 현재 진행 중인 연구 방향
* 관련 논문 및 프로젝트 연결

### 5.5 Publications

연구실 논문 성과를 정리하는 페이지이다.

표시 정보 예시:

* 논문 제목
* 저자
* 게재처
* 연도
* PDF / DOI / Project / Code / Video 링크

정렬 방식 예시:

* 연도별 정렬
* 대표 논문 강조

### 5.6 Projects

진행 중이거나 완료된 연구 프로젝트를 정리하는 페이지이다.

표시 정보 예시:

* 프로젝트명
* 프로젝트 개요
* 참여 인원
* 관련 논문
* 데모 또는 결과물 링크

### 5.7 News / Notice

게시글 형태로 운영되는 영역으로, 홈페이지의 최신성을 유지하는 핵심 메뉴이다.

운영 방식:

* Notice: 공지사항, 모집, 일정 안내
* News: 논문 게재, 수상, 발표, 행사 소식

초기 운영 시에는 두 영역을 통합하여 **News & Notice** 형태로 운영할 수 있다.

### 5.8 Join Us

연구실 지원 희망자를 위한 안내 페이지이다.

주요 구성 요소:

* 모집 대상
* 관심 분야
* 우대 역량
* 지원 방법
* FAQ
* 문의 이메일

### 5.9 Contact

연락처 및 위치 정보를 제공하는 페이지이다.

주요 구성 요소:

* 주소
* 건물 및 호실
* 이메일
* 전화번호
* 지도 또는 위치 안내

## 6. 관리자 페이지 상세 구성

### 6.1 Login

관리자만 접근 가능한 인증 페이지이다.

구성 요소:

* 아이디
* 비밀번호
* 로그인 버튼

접근 방식:

* 별도 관리자 URL 사용 (예: `/admin`)
* 로그인 성공 시 관리자 대시보드로 이동

### 6.2 Dashboard

관리자 로그인 후 최초로 진입하는 화면이다.

주요 기능:

* 최근 작성 게시글 확인
* 최근 수정 내역 확인
* 자주 사용하는 관리 메뉴 바로가기

### 6.3 게시글 관리

공지 및 소식을 등록·수정·삭제하는 기능을 제공한다.

주요 기능:

* 게시글 목록 조회
* 작성 / 수정 / 삭제
* 공개 여부 설정
* 상단 고정 여부 설정
* 카테고리 지정

입력 항목 예시:

* 제목
* 작성일
* 카테고리
* 본문
* 첨부파일
* 썸네일

### 6.4 구성원 관리

People 페이지의 정보를 관리하는 기능이다.

주요 기능:

* 구성원 추가 / 수정 / 삭제
* 소속 그룹 변경
* 노출 순서 조정
* 프로필 사진 및 링크 등록

### 6.5 연구 / 프로젝트 관리

Research 및 Projects 페이지에 노출되는 정보를 관리한다.

주요 기능:

* 연구 분야 설명 수정
* 프로젝트 등록 / 수정 / 삭제
* 대표 프로젝트 지정
* 관련 링크 연결

### 6.6 논문 관리

Publications 페이지의 논문 정보를 관리한다.

주요 기능:

* 논문 등록 / 수정 / 삭제
* 연도 분류
* 게재처 입력
* 외부 링크 연결
* 대표 논문 지정

### 6.7 메인 화면 관리

Home 페이지에 노출되는 주요 콘텐츠를 손쉽게 변경할 수 있도록 한다.

주요 기능:

* 메인 문구 수정
* 대표 이미지 변경
* 최신 소식 노출 설정
* 대표 논문 / 프로젝트 선택

### 6.8 관리자 계정 설정

관리자 계정에 대한 기본 설정 기능을 제공한다.

주요 기능:

* 비밀번호 변경
* 계정 정보 확인
* 예비 관리자 추가 가능 여부 검토

## 7. 권한 및 접근 정책

### 7.1 사용자 권한 구분

* **일반 방문자**: 공개 페이지 열람만 가능
* **관리자**: 관리자 페이지 로그인 후 등록/수정/삭제 가능

### 7.2 권한 운영 방안

* 초기에는 관리자 계정 1개를 기본으로 운영한다.
* 필요 시 예비 관리자 1명을 추가할 수 있다.
* 일반 사용자 대상 회원가입 기능은 제공하지 않는다.

### 7.3 로그인 정책

* 관리자 전용 URL을 통해 접근한다.
* 인증된 관리자만 관리 기능을 사용할 수 있다.
* 공개 홈페이지에는 별도 로그인 기능을 두지 않는다.

## 9. 기대 효과

* 연구실 정보를 체계적이고 일관된 방식으로 제공할 수 있다.
* 외부 방문자와 지원자가 필요한 정보를 빠르게 확인할 수 있다.
* 관리자 1인 중심의 단순한 운영 구조로 유지보수 부담을 줄일 수 있다.
* 홈페이지의 최신성과 신뢰도를 확보할 수 있다.

## 10. 제안 사항

초기 구축 단계에서는 기능을 과도하게 확장하기보다, 다음 요소를 중심으로 우선 구현하는 것이 적절하다.

1. 공개 홈페이지 기본 메뉴 구성
2. 관리자 로그인 및 게시글 관리 기능
3. People / Publications / Home 주요 정보 수정 기능
4. News & Notice 중심의 최소 운영 체계 구축

이후 실제 운영 과정에서 필요성이 확인되는 기능에 한해 점진적으로 확장하는 방향을 권장한다.
